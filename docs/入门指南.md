# BactTrading 入门指南

## 1. 准备环境

### 1.1 安装依赖

```bash
pip install -r requirements.txt
cd frontend
npm install
cd ..
```

### 1.2 配置 `.env`

```bash
cp .env.example .env
```

最常用配置：

```env
APP_HOST=0.0.0.0
APP_PORT=8051
EXCHANGE_PROXY_URL=

BINANCE_API_KEY=
BINANCE_API_SECRET=
OKX_API_KEY=
OKX_API_SECRET=
OKX_API_PASSPHRASE=
```

如果你只是读取已有 CSV，可以先不填 API Key。

## 2. 启动方式

### 2.1 构建前端

```bash
cd frontend
npm run build
cd ..
```

### 2.2 启动服务

```bash
python start_server.py
```

浏览器访问：

```text
http://127.0.0.1:8051
```

## 3. 页面怎么用

### 3.1 左侧控制面板

你可以配置：

- 交易所
- 数据文件
- 交易对
- 周期
- 开始/结束日期
- 最少交易次数
- 指标开关

点击 `加载图表` 后，会从 CSV 和交易所数据中拼出完整复盘视图。

### 3.2 图表工作区

新页面支持：

- K 线主图
- 成交量 pane
- RSI pane
- MACD pane
- EMA20 / 布林带叠加
- 十字线 legend
- 右键菜单
- 工具条
- 键盘快捷键

常用快捷键：

- `1-6` 切周期
- `F` 适应内容
- `L` 跳到最新
- `E` 切 EMA
- `B` 切布林带
- `I` 切 RSI
- `M` 切 MACD
- `T` 切交易标记

### 3.3 仓位导航

页面下方保留了完整仓位复盘能力：

- 上一笔 / 下一笔
- 序号跳转
- 自动聚焦该仓位时间范围
- 仓位信息卡

## 4. 重建仓位 CSV

### 4.1 命令行

```bash
python getPosition.py -e binance -s 2025-07-18 -n 2025-08-02
```

### 4.2 页面内触发

左侧“重建仓位”面板会调用后端 API，执行同样的任务，并把新 CSV 写回 `data/`。

## 5. 常见问题

### 5.1 页面打不开

- 确认你已经执行过 `cd frontend && npm run build`
- 确认 `python start_server.py` 正在运行
- 检查 `.env` 中的 `APP_PORT`

### 5.2 图表加载不到数据

- 检查选中的时间范围内是否存在该交易对
- 检查 CSV 是否包含该 symbol
- 如果走交易所 API，检查代理和 API Key

### 5.3 仓位重建失败

- 检查 API Key 是否有效
- 检查代理是否可用
- 检查交易所是否允许当前查询时间范围
